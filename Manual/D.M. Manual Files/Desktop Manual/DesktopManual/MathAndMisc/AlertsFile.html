Alerts: File Errors


The following is a listing of our translations of the error messages returned by the Macintosh if an error occurs when working with a file:
  • err -33 File Directory is full.
  • err -34 Volume is full.
  • err -35 No such volume.
  • err -36 I/O Error!
  • err -37 Bad file or volume name.
  • err -38 File is not open.
  • err -39 Logical EOF during file read.
  • err -40 Attempt to position before start of file.
  • err -42 Too Many files open.
  • err -43 File not found.
  • err -44 Volume is locked by a hardware setting.
  • err -45 File is locked.
  • err -46 Volume locked by software flag.
  • err -47 File is busy.
  • err -48 Duplicate file name.
  • err -49 File already open with write permission.
  • err -50 Volume is not properly specified.
  • err -51 Ref num specifies nonexistent access path.
  • err -52 Error during GetFPos.
  • err -53 Volume is not on line.
  • err -54 Attempt to open a locked file writing.
  • err -55 Specified volume is already mounted and on-line.
  • err -56 Specified drive number matches no known drive.
  • err -57 Not a Macintosh volume.
  • err -58 External file system error.
  • err -59 Rename was unsuccessful.
  • err -60 Master directory block is bad, re-initialize the volume.
  • err -61 File not open for writing.
  • err -108 Not enough room in heap zone.
  • err -120 Directory not found.
  • err -121 Too many working directories open.
  • err -122 Attempted to move into offspring.
  • err -123 Attempt to do an HFS operation on a non HFS volume.
  • A file is already open...OOPS
  • File Ptr and requested size do not match. The data is not consistent...ReadDataBlock error
